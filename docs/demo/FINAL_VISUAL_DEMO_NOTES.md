# CustosOps Final Visual Demo Notes

## Current visual baseline

```text
custosops-v0.30.9-final-visual-demo-package
```

## Demo objective

Show CustosOps as a local-first defensive security evidence console for security hygiene, posture review, and evidence/report generation.

## Recommended demo path

```text
1. Start on Overview.
2. Explain that all evidence is local/read-only.
3. Show KPI row, severity distribution, module health, and findings by module.
4. Show recent runs, archive/report summary, and top priority findings.
5. Open Endpoint, DNS, Windows Events, IIS/Application, Reports, Archive, Run History, and Redaction.
6. Explain redaction, local processing, and report generation.
```

## What to emphasize

```text
- Defensive/read-only posture.
- Evidence collection/import and local processing.
- SMB security hygiene angle.
- Practical support/security reporting.
- Redaction-aware outputs.
- Repeatable proof gates and UI smoke validation.
```

## What not to claim

```text
- Not a SIEM.
- Not an EDR.
- Not a vulnerability scanner.
- Not an MDR/MSSP platform.
- Not an offensive testing tool.
- No active remediation.
```
