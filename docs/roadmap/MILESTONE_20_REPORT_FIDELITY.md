# Milestone 20 - Report Fidelity v0.1

## Goal

Make report artifacts as trustworthy as the evidence workspaces.

## Fixed

- Endpoint reports infer the endpoint name from finding affected assets when raw evidence is restored from session state.
- DNS reports derive a useful record count from loaded findings when raw DNS record lists are not available.
- Restored/session evidence is labeled clearly.
- Application Log HTML reports now render as proper HTML instead of Markdown-looking text wrapped in paragraphs.

## Added

- Regression tests for endpoint session fallback report names.
- Regression tests for DNS derived record counts.
- Regression tests for App Log HTML rendering.

## Why it matters

Reports are the final operational artifact. They must not look weaker or less precise than the evidence workspace.

## Safety

No remediation was added.

The fallback logic does not invent technical observations. It only derives metadata from findings already generated by CustosOps.
